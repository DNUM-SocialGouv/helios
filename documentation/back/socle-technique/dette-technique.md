# Dette technique

## Présentation

La dette technique regroupe les éléments techniques qui nécessitent une refactorisation ou une amélioration mais qui ne sont pas prioritaires fonctionnellement.

## Éléments de dette identifiés

### Architecture

- Optimisation des requêtes de comparaison pour de grands volumes
- Amélioration de la gestion du cache
- Refactorisation de certains use-cases complexes

### Base de données

- Optimisation des index pour améliorer les performances de recherche
- Nettoyage des données obsolètes
- Amélioration de la gestion des migrations

### Code

- Modernisation de certaines dépendances
- Amélioration de la couverture de tests
- Refactorisation de composants frontend complexes

## Suivi

La dette technique est suivie dans l'outil de gestion de projet (Jira) et priorisée selon l'impact et les risques.
